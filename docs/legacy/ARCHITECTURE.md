# WEALTH Architecture — v2.1.0 (MCP Federation)
# DITEMPA BUKAN DIBERI

## 1. Core Thesis: Sovereign Federation
WEALTH is a **local-first financial governance host** that orchestrates a modular federation of **MCP (Model Context Protocol)** servers. This separation ensures that domain-specific intelligence remains bounded, while the Host retains ultimate policy control and F1-F13 enforcement.

---

## 2. The 4-Tier Stack

### Tier 0: The Host (`/host`)
The "Brain" of the system. It handles:
- **Policy Engine:** Enforcement of F1-F13 floors.
- **888_GATE:** Human-in-the-loop authorization for all destructive/sensitive tools.
- **Sovereign Dashboard:** The primary Nexus UI.

### Tier 1: Domain Intelligence (`/mcp`)
Specialized MCP servers that expose tools, resources, and prompts:
1. **`wealth-finance-mcp`**: The Truth Layer (Net Worth, Cashflow, Runway).
2. **`wealth-market-mcp`**: The Analysis Layer (Stock Screening, Macro Risk).
3. **`wealth-energy-mcp`**: The Scarcity Layer (Grid Stress, Fuel Monitoring).
4. **`wealth-food-mcp`**: The Bread Layer (Inflation, Supply Shock Monitoring).
5. **`wealth-trust-mcp`**: The Audit Layer (Lineage, Provenance, Drift Status).
6. **`wealth-risk-mcp`**: The Scenario Layer (Shock Tests, Livelihood Coupling).

---

## 3. Design Rules
- **Host Decides Policy:** Only the Host can seal a 999 verdict.
- **Apps Provide Facts:** MCP apps provide analysis but have no sovereign authority.
- **Approval Classes:** 
    - `read-only`: Automatic execution.
    - `sensitive`: Requires 888_GATE hold.
    - `destructive`: Requires F13 Human Veto.
